#Введение
Языки программирования разделяются на 3 вида
- Компилируемые: С, C++, Pascal.

    Код компилируемых языков перед использованием необходимо пропустить
    через компилятор и получить исполняемый файл(например .exe) и только потом запускать.

    > В случае появлении ошибки продолжает компиляцию до конца и пытается выполнить программу полностью.

<br>

- Интерпритируемые: VBScript, JavaScript, Python, PHP.

    В реальном времени переводит код в комманды микро процессора
    при помощи специальной заготовленной программы.
    
    > В с лучае появлении ошибки не продолжает программу.

<br>

- Условно компилируемые: C# и остальные языки .Net, TypeScript, Java, Java-машины.

    Для таких языков компиляция проходит в два этапа.
    (Возьмем для примера С#)Язык С# переводится в CIL - Common Intermidate Language
    которая передается на вход JIT - Just In Time компилятору. Все эти выкрутасы
    делаются ради того чтоб не заботится о типи микро процессора который запускает исходный файл.


<br>

## Работа с строками
Название метода|Применение|Дополнительно
--|----|----
length()|возвращает длинну строки
split(" ")|Возвращает массив элементов строки которые разделены указанным символом|Если указать "" то вернет массив посимвольно
charAt(7)|Возвращает символ который находится в строке
toUpperCase()|Все символы в строке становятся заглавными
toUpperCase()|Все символы в строке становятся строчными
indexOf("banana")|Возвращает позицию первого символа в слове который равен указанной строке|Если слово находится в конце строки то возвращает 1
lastIndexOf("banana")|Возвращает позицию последнего символа в слове который равен указанной строке
includes("banana")|Возвращает true/false при поиске указанного слова в строке
startsWith("x")|Возвращает true/false при при поиске первого слова в строке
endsWith("x")|Возвращает true/false при при поиске последнего слова в строке
slice(start, end)|Возвращает слово которое находится от start до end|Если start > end возващает пустую строку, можно передовать отрицательные значения
substring(start, end)|Возвращает слово которое находится между start и end|Отрицательные значения = 0
substr(start, end)|Возвращает length() символов начиная от start|Можно передовать отрицательные значения
codePoint(7)|Возвращает код символа на указанной позиции
fromCodePoint(90)|Возвращает символ с данным кодом
trim()|Обрезает пробелы с коцов строки
repeat(7)|Вернет строку с повторением строки указанное колличество раз
replace(target, value)|Заменяет первое попавшийся target указанным value
replaceAll(target, value)|Заменяет каждый target указанным value

## Работа с массивами
Название метода|Применение|Дополнительно
---|----|----
forEach(element => console.log(element.name))|Применяет функцию к каждому элементу массива
pop()|Удаляет элемент из конеца массива
push()|Добавляет элемент в конец массива
shift()|Удаляет элемент в начале, сдвигая очередь
unshift()|Добавляет элемент в начало массива
concat()|Принемает массив, возвращает массив из двух обьеденных
reduce((total, value) => total * value)|Принемает функцию, которая имеет аккумулятор и значение, он применяет функцию к акумулятору и каждому значению|От автора: Честно говоря несовсем понятная штука, но она действительно имеет свое применение и игратся сней прикольно
some(test => test === "banana")|Логический, проверяет удовлетворяет ли какой либо элемент массива условию задданой в передоваемой фунциии. Ели хотябы один элемент подходит то true
every(test => test === "banana")|Как some() но возвращает true только если все элементы соостветствуют условию
filter(element => element.name === "Banana")|Принемает массив, возвращает массив из элементов которые соответствовали заданному условию
map(x => x * x)|Принемает функцию, возвращает массив с результатом функции для каждого элемента массива
flat()|Принемает массив, возвращает массив с сглаженной вложенностью внутри массива на 1 уровень
find(element => element.id === 3)|Возвращает первый попавшийся элемент массива удовлетворяющий условие
findIndex(element => element.id === 3)|Принемает фунцию, возвращает индекс элемента удовлетворяющего условие|Если не один не удовлетворяет, возвращает -1
sort((a, b) => a - b)|Принемает функцию, возвращает отсортированный массив
fill(0, 2, 4)|Заполняет указанные элементы массива начальным
reverse()|Принемает массив, возвращает отраженный массив
includes("banana")|Логический, проверяет существует ли указанные элемент массива
flatMap(arr => arr * 10)|Обьеденяет в себе map() и flat() и применяет их в данной последовательности

## Медленные и быстрые методы
Быстрые методы(pop, push ...), медленные методы(shift, unshift ...)

Почему "медленные методы" медленные?

Предположим у нас есть массив из 4 элементов
и мы делаем shift(), нулевой элемент массива удаляется затем элемент на позиции 1 переходит на нулевую, элемент с позиции 2 на 1, элемент с позиции 3 на 2 и так далее до конца массива. Чем больше массив тем дольше выполнение. Следует предположить что не только shift и unshift являются медленными методами
